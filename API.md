# API

Base path: `/api/v1`. Current identity endpoints: `POST /auth/register`, `POST /auth/login`, `POST /auth/logout`, `GET /me`, `GET /organizations/:id`, `GET|POST /users`, and `GET /audit`. Asset endpoints are `GET|POST /assets` and `GET|PATCH|DELETE /assets/:id`. Asset-area endpoints are `POST /asset-areas` and `PATCH|DELETE /asset-areas/:id`, with `GET /assets/:id/areas`; equipment endpoints are `POST /equipment` and `PATCH|DELETE /equipment/:id`, with `GET /assets/:id/equipment`. Vessel endpoints are `GET|POST /vessels`, `PATCH|DELETE /vessels/:id`, and `GET|POST /vessels/:id/positions`; voyage endpoints are `GET|POST /voyages` and `PATCH|DELETE /voyages/:id`. Personnel endpoints are `GET|POST /people`, `PATCH|DELETE /people/:id`, `GET|POST /competencies`, `PATCH|DELETE /competencies/:id`, `GET|POST /people/:id/competencies`, `GET /assets/:id/pob`, `POST /pob` and `POST /pob/:id/disembark`. Operational endpoints are `GET|POST /activities`, `PATCH|DELETE /activities/:id`, `GET /activities/timeline`, and `GET|POST /activity-dependencies`; dependencies are rejected when they would create a cycle. `GET /map/operational` returns tenant-scoped GeoJSON for assets and latest vessel positions. All records are tenant-scoped. Administrators and operations coordinators can create or update them, while only administrators can delete them. An area cannot be deleted while it has child areas or equipment. `GET /health` is liveness; `GET /ready` checks PostgreSQL. Authentication is a secure cookie and all protected endpoints require it.
